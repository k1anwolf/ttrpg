# Анализ интерфейса "Боевой дэшборд TTRPG — Полная версия"

## Общее описание
Это комплексный веб-интерфейс для управления боями в настольных ролевых играх (TTRPG), представляющий собой полнофункциональную систему трекинга боевых encounters с расширенными возможностями управления состоянием участников, эффектами и сохранениями.

---

## Детальная структура и функционал

### 1. **Верхняя панель (Header)**
- **Заголовок**: "Боевой дэшборд TTRPG — Полная версия"
- Статичная навигационная панель с основным названием приложения

### 2. **Основная рабочая область (Wrap)**
Разделена на три основные секции:

#### **A. Левая панель управления**
**Трекер инициативы:**
- Отображение списка участников в порядке инициативы
- Цветовое кодирование: игроки (синий), НПС (красный), боссы (розовый)
- Визуальное выделение активного участника
- Показ текущего значения инициативы

**Управление ходом:**
- Кнопки навигации (предыдущий/следующий ход)
- Завершение раунда
- Отмена последнего действия
- Сброс раундов (новый бой)

**Счётчик раундов:**
- Отслеживание текущего раунда
- Кнопка обнуления для нового боя

**Управление участниками:**
- Добавление новых участников
- Клонирование существующих
- Сохранение/загрузка состояния в браузере

**Система сохранений:**
- Быстрое сохранение
- Менеджер сохранений с поиском
- Экспорт/импорт в файлы
- Управление пакетами сохранений

**Дополнительные менеджеры:**
- Менеджер шаблонов персонажей и существ
- Менеджер эффектов
- Настройки отдыха
- Экспорт/импорт кампаний

#### **B. Центральная рабочая область**
**Панель управления боем:**
- Кнопки короткого/длинного отдыха
- Доступ к истории действий

**Основная таблица участников:**
Колонки таблицы:
1. **Порядковый номер** - позиция в инициативе
2. **Инициатива** - редактируемое значение
3. **Имя** - редактируемое поле
4. **Фракция** - выбор из: Игрок/НПС/Босс
5. **HP** - 
   - Для обычных участников: полоса здоровья с визуальной индикацией
   - Для боссов: отображение нанесенного урона
6. **MP** - полоса маны/ресурса
7. **Характеристики** - 6 основных характеристик с редактором
8. **AC/Навыки** - класс брони и список навыков
9. **Атаки** - список атак с управлением CD
10. **Умения** - список способностей
11. **Заклинания** - список заклинаний
12. **Состояния** - активные эффекты и статусы
13. **Управление** - панель быстрых действий

**Блок быстрых действий:**
- Быстрый урон/лечение (-5/-10 HP, +5/+10 HP)
- Запуск действий: атака, умение, заклинание
- Ручное перемещение
- Удаление участника

**Система статусов:**
- Визуальные чипы с отображением длительности
- Управление смертельными спасбросками
- Инструменты добавления/удаления статусов

#### **C. Лог событий**
- Хронологическая запись всех действий в бою
- Автоматическое обновление и прокрутка

---

### 3. **Модальные окна и системы**

#### **A. Система сохранений**
**Менеджер сохранений:**
- Просмотр всех сохранений с поиском
- Создание новых сохранений
- Импорт/экспорт отдельных файлов и пакетов
- Метаданные: название, описание, тип, версия

**Типы сохранений:**
- Только бой
- Полное (бой + шаблоны)
- Только шаблоны

#### **B. Система шаблонов**
**Категории шаблонов:**
- **Персонажи** - игровые персонажи
- **Враги** - обычные противники
- **Боссы** - уникальные сильные противники
- **Заклинания** - магические способности
- **Умения** - специальные способности
- **Атаки** - боевые приемы
- **Эффекты** - статусные эффекты

**Функционал шаблонов:**
- Создание/редактирование/удаление
- Импорт/экспорт в JSON
- Использование шаблонов для быстрого создания участников
- Сохранение текущих участников как шаблонов

#### **C. Система действий и эффектов**
**Редактор действий:**
- Настройка названия, типа, КД, ограничений
- Детальное описание эффектов

**Система эффектов:**
- **Типы эффектов:**
  - Урон
  - Лечение
  - Восстановление MP
  - Наложение статусов
  - Снятие статусов
  - Толчок, захват, положение ниц
  - Воскрешение

**AoE-режим:**
- Массовое применение эффектов
- Единые настройки для всех целей

#### **D. Менеджер эффектов**
- Библиотека готовых эффектов
- Быстрое применение к целям
- Управление шаблонами эффектов

#### **E. Настройки отдыха**
- Конфигурируемые проценты восстановления:
  - Короткий отдых: HP/MP %
  - Длинный отдых: HP/MP %

#### **F. История действий**
- Хронологический журнал всех совершенных действий
- Возможность очистки истории
- Детализация действий с временными метками

---

### 4. **Технические особенности**

#### **A. Визуальное оформление**
- Темная тема с акцентными цветами
- Цветовое кодирование фракций
- Анимации (пульсация, slide-in)
- Визуальные индикаторы состояния:
  - Цветовые градации HP (зеленый-желтый-красный)
  - Полосы прогресса HP/MP
  - Статусные чипы с таймерами

#### **B. Система данных**
**Структура участника:**
```javascript
{
  id, name, initiative, faction,
  hpMax, hpCurr, mpMax, mpCurr,
  ac, skills, characteristics,
  attacks[], abilities[], spells[], statuses[]
}
```

**Система состояний:**
- Отслеживание длительности в раундах/ходах
- Автоматическое уменьшение длительности
- Визуальное отображение оставшегося времени

#### **C. Локальное хранение**
- Сохранение состояния в localStorage
- Резервное копирование в файлы
- Восстановление полного состояния приложения

---

### 5. **Игровая механика**

#### **A. Система боя**
- Сортировка по инициативе
- Последовательные ходы
- Автоматическое завершение раундов
- Отслеживание общего количества ходов

#### **B. Система смерти**
- Механика спасбросков от смерти
- Визуализация успехов/провалов
- Автоматическая стабилизация/смерть
- Возможность воскрешения

#### **C. Управление ресурсами**
- HP/MP с визуальными полосами
- Система отдыха с настраиваемым восстановлением
- КД способностей и заклинаний

---

### 6. **Расширенные возможности**

#### **A. Импорт/Экспорт**
- Отдельных участников
- Целых боёв
- Библиотек шаблонов
- Полных кампаний

#### **B. Кастомизация**
- Настраиваемые характеристики
- Гибкая система действий
- Конфигурируемые эффекты
- Адаптируемые настройки отдыха

#### **C. Демо-режим**
- Предустановленные демо-данные
- Примеры персонажей, врагов, боссов
- Готовые шаблоны способностей и эффектов

---

## Преимущества системы

1. **Полнофункциональность** - охватывает все аспекты управления боем
2. **Гибкость** - адаптируется под различные TTRPG системы
3. **Удобство использования** - интуитивный интерфейс с быстрыми действиями
4. **Надёжность** - комплексная система сохранений и резервного копирования
5. **Расширяемость** - возможность импорта/экспорта и создания библиотек

Этот интерфейс представляет собой профессиональный инструмент для мастеров игр, значительно упрощающий управление сложными боями в настольных ролевых играх.
